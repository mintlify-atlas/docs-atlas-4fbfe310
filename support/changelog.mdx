  </Accordion>

  <Accordion title="2.0.0">
    - Added a password recovery flow so users can regain access to their account if they forget their password.
    - Users can now verify whether their email was registered without their knowledge and delete the account if it wasn't them.
    - Updated the app logo.
    - Generated a token-based account deletion page.
    - Generated a token-based password reset page.
    - Integrated Nodemailer for sending account verification and recovery emails.
    - Integrated the Node.js `crypto` module for secure token generation.
    - Improved routing and frontend code optimization.
    - Bug fixes.
  </Accordion>

  <Accordion title="1.1.1">
    - Fixed a bug when logging in with an email that doesn't exist.
    - Fixed a bug when registering with an email that is already in use.
    - Preparation for upcoming features and improvements.
  </Accordion>

  <Accordion title="1.1.0">
    - Added backend error indicators for clearer failure feedback.
    - Reports can now be deleted from the admin panel (for reports that are resolved or no longer relevant).
    - Improved visibility and usability of the admin login page.
  </Accordion>

  <Accordion title="1.0.1">
    - Added download links for the new app versions.
  </Accordion>

  <Accordion title="1.0.0 — Initial stable release">

    - Web app ready, pending new app version uploads and bug fixes.
