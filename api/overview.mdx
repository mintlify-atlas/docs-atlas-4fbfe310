The CloudBook API is a REST API. All requests and responses use JSON. Use it to build integrations, automate workflows, or embed CloudBook functionality into your own application.

**Base URL**

```
https://list-app.vercel.app/api
```

## Request format

Every request that sends data must include the `Content-Type: application/json` header and a JSON body.

```bash
curl -X POST https://list-app.vercel.app/api/users/login \
  -H "Content-Type: application/json" \
  -d '{"email": "you@example.com", "password": "yourpassword"}'
```

## Authentication

Authenticated endpoints require a JWT token in the `Authorization` header. You receive this token when you register or log in.

```bash
curl https://list-app.vercel.app/api/notes \
  -H "Authorization: <your-token>"
```

See [Authentication](/api/authentication) for details on how to obtain and use tokens.

## Internal API key layer

All API routes are protected by an internal `api-key` middleware. This is a server-side protection layer — it validates a shared secret passed in the `api-key` request header against an environment variable. This mechanism is not intended for direct use by external clients. External clients authenticate using the JWT token described above.

## Error responses

When a request fails, the API returns a JSON object with a `message` field and, where applicable, an `error` field with additional detail.

| Field     | Type   | Description                                      |
|-----------|--------|--------------------------------------------------|
| `message` | string | Human-readable description of what went wrong.  |
| `error`   | string | Machine-readable error detail or upstream message. Present on most 4xx and 5xx responses. |

Example error response:

```json
{
  "message": "El correo no está registrado",
  "error": "Not found"
}
```

## Status codes

| Code | Meaning                                                                 |
|------|-------------------------------------------------------------------------|
| `200` | Request succeeded.                                                     |
| `204` | Request succeeded with no response body (used for preflight OPTIONS).  |
| `400` | Required fields are missing from the request body.                     |
| `401` | Authentication failed — missing or invalid credentials.                |
| `403` | Required fields are missing from the request body (used by some endpoints as an alternative to 400). |
| `404` | The requested resource was not found.                                  |
| `408` | The request timed out.                                                 |
| `409` | Conflict — for example, attempting to register an email already in use. |
| `500` | An unexpected server-side error occurred.                              |
